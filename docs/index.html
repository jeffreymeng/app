Intro

The Jeffrey Viewer API provides a simple URL-based file viewing experience. It supports many formats and provides the user with a beautiful interface to view them. After viewing the file, the page will link back to your url.

Api Limit

There is currently no API limit, though if you are going to use the API heavily, please contact us first - viewer jeffkmeng.com.

Getting Started

All file types utilize the same url, referred to as an open URL:

https://viewer.jeffkmeng.com/open?parametershere

The open URL(above) is the base of the path you will redirect to, regardless of parameters, but this is most likely not going to be the URL your users see. While our links for each individual video type may change at any time, our open url will (most likely) always be supported. All URL's have base and type-specific file parameters. Base file parameters apply to all file types, while type-specific parameters apply only to a certain data type. There are required base and type-specific file parameters, as well as optional parameters for both.

File Types

Video

Mode: video

Supported Video Types:
  • video/mp4
  • video/webm
  • YouTube
  • Vimeo

Video Specific Url Parameters:
Name Description Accepted Types(after parsing) Default
url The URL of the video (of a supported video type) file to play. String Required
type The media type of the video. It must exactly match a supported filetype above. String. Required
location Optional metadata for the location of the video. If the value is false, it is not displayed. Otherwise, it is displayed with the website of the file automatically appended. String or boolean(value of false only). Location is not displayed("" and the website of the file is not appended)
altTracks Alternative video URL and types. If either the url or type parameters are omitted or fail, then the first(0th) object inside the array's url and type keys will be used. If those fail, then the next element's keys will be used, and so on until there are no more elements. Array of objects. Each object must have url and type keys with values that follow the corresponding parameter above . []
